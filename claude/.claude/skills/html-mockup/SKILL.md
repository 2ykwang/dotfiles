## Instructions

사용자의 요청을 **인터랙티브한 단일 HTML 파일**로 시각화한다.

### 핵심 원칙

1. **단일 self-contained HTML** — inline CSS + JS. 외부 의존성(CDN, 라이브러리) 없음
2. **반드시 인터랙티브** — 정적 이미지가 아닌, 실제로 조작 가능해야 한다
3. **다크/라이트 모드 토글** — 우측 상단에 토글 버튼 배치. 기본값은 다크
4. **파일명**: `mockup-[kebab-case-이름].html` — 프로젝트 루트에 저장

### 퀄리티 기준

결과물은 다음 기법들을 적절히 활용하여 깔끔하고 일관적인 시각화를 만든다:

- **렌더링**: SVG 또는 Canvas 기반으로 선명하게 표현. 시각화 유형에 맞는 것을 선택
- **인터랙션**: 드래그 이동, 탭/클릭, 호버 하이라이트, 토글, 스와이프 등 맥락에 맞는 조작
- **트랜지션**: 상태 전이, 화면 전환, 요소 등장/퇴장에 부드러운 애니메이션
- **레이아웃**: 요소 간 겹침 방지, 적절한 간격, 반응형 고려
- **디바이스 프레임**: 모바일/태블릿 UI 목업 시 실제 기기 형태의 프레임 사용
- **연결선**: 다이어그램/ERD에서 노드 드래그 시 관계선 자동 추적
- **범례/라벨**: 여러 화면이면 나란히 배치 + 라벨, 다이어그램이면 범례 포함
- **placeholder 데이터**: 프로젝트 맥락에 맞는 현실적인 데이터 사용

### 절차

1. 인자와 대화 맥락에서 **무엇을 시각화할지** 파악한다. 모호하면 질문한다.
2. 프로젝트 맥락이 필요하면 관련 코드/문서를 읽어 실제 데이터와 구조를 반영한다.
3. 시각화 유형에 맞는 인터랙티브 HTML을 생성한다.
4. `open mockup-[name].html` 으로 열 수 있음을 안내한다.
